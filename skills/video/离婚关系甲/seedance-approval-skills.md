# 即梦（Seedance）提示词审核通过完全技能指南

## 📋 概述

本指南整合了本地知识库和实战经验，提供即梦提示词无法过审的全面解决方案，涵盖敏感词处理、人脸审核、内容安全等所有相关技能。

## 🔍 审核失败原因分析

### 1. 人脸审核失败（最常见）
**平台限制**：即梦明确禁止真实人脸
- 上传图片/视频中含真实人脸 → 直接拦截
- 生成视频中人脸过于写实 → 审核不通过
- 肖像权风险 → 平台合规要求

### 2. 敏感内容审核
**内容安全机制**：
- 暴力、血腥、恐怖内容
- 政治敏感、历史敏感话题
- 封建迷信、灵异内容
- 社会负面、低俗内容

### 3. 技术限制
**平台技术约束**：
- 文件数量超限（>12个）
- 文件格式不支持
- 文件大小超限
- 视频时长超限（>15秒）

### 4. 内容质量
**生成质量审查**：
- 画面混乱，无法识别内容
- 严重逻辑错误
- 技术故障导致的异常画面

#### 2. 内容安全原则

**四大处理原则**：
1. **核心需求保留原则** - 保留冲突、对决、成长等核心，调整表现形式
2. **意象表达原则** - 用意象代替直白描述（暴力→力量对决，恐怖→紧张悬疑）
3. **科技奇幻化原则** - 将迷信玄幻转化为科幻（魔法→科技/超能力）
4. **文化适应性原则** - 考虑不同文化背景，避免文化冒犯

#### 3. 平台特定要求

**抖音/快手**：
- 避免低俗、恶俗内容
- 正能量导向
- 符合社会主义核心价值观
- 青少年模式安全

**YouTube**：
- 避免版权音乐
- 注意社区准则
- 考虑国际观众
- 多语言字幕友好

**B站**：
- 符合ACG文化
- 注意弹幕礼仪
- 尊重UP主文化
- 社区氛围维护

## 🚀 实战工作流程

### 阶段1：风险评估（5分钟）
```
1. 内容敏感度分析
   - 是否有真实人脸需求？
   - 是否有敏感主题？
   - 是否有暴力/恐怖元素？

2. 安全等级确定
   - 一级安全：完全抽象
   - 二级安全：风格化处理
   - 三级安全：特征简化
   - 风险区域：写实风格

3. 处理方案选择
   - 人脸处理方案
   - 敏感词替换方案
   - 技术参数设置
```

### 阶段2：提示词设计（10分钟）
```
1. 基础结构搭建
   【场景】+【人物】+【动作】+【运镜】+【情绪】+【风格】

2. 安全元素添加
   - 风格化关键词
   - 安全声明
   - 技术参数

3. 敏感词检查
   - 自动扫描敏感词
   - 手动替换高风险词汇
   - 添加安全替代词
```

### 阶段3：生成测试（5分钟）
```
1. 小样生成
   - 生成15秒测试视频
   - 检查人脸识别度
   - 评估审核风险

2. 优化调整
   - 根据结果调整参数
   - 加强风格化处理
   - 优化敏感词替换

3. 最终确认
   - 确认安全等级达标
   - 保存成功模板
   - 记录优化经验
```

### 阶段4：批量生产（按需）
```
1. 模板应用
   - 使用成功模板
   - 批量生成内容
   - 保持一致性

2. 质量检查
   - 批量审核检查
```

```python
        "巫师": "神秘黑袍法师",
        "魔法": "能量操控",
        "鬼魂": "光影化身",
```

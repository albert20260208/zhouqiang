## 👥 多Agent协作模式 (FilmAgent论文)

### 4个核心角色及职责

| 角色 | 职责 | 具体工作 |
|------|------|----------|
| **Director (导演)** | 整体把控+最终决策 | 设定角色档案、制定大纲、审核剧本、解决冲突 |
| **Screenwriter (编剧)** | 写剧本+动作设计 | 写对白、指定角色位置和动作、根据反馈修改 |
| **Actor (演员)** | 角色代言 | 微调台词使其符合角色性格、与导演沟通 |
| **Cinematographer (摄影师)** | 镜头设计 | 为每句台词选择镜头、与同行辩论、确保镜头合适 |

### 两大协作算法 (核心!)

#### 1. Critique-Correct-Verify (批评-修正-验证)
**用于: 剧本写作阶段**

```
步骤1: Action Agent生成剧本
         ↓
步骤2: Critique Agent审核,写出批评意见
         ↓
步骤3: Action Agent根据批评修正剧本
         ↓
步骤4: Critique Agent验证是否解决问题
         ↓
通过 → 进入下一阶段
未通过 → 回到步骤2 (最多3轮)
```

**实际应用:**
- 导演(Critique) + 编剧(Action): 审核剧情连贯性
- 导演(Critique) + 演员(Action): 审核对白是否符合角色

#### 2. Debate-Judge (辩论-裁决)
**用于: 摄影设计阶段**

```
步骤1: 摄影师A和B各自独立选择镜头
         ↓
步骤2: 双方互相评价对方的选择
         ↓
步骤3: 辩论各自方案的优劣
         ↓
步骤4: 导演(Judge)总结辩论,做最终决定
```

**实际案例:**
```
摄影师A: 建议用中景,捕捉肢体语言
摄影师B: 建议用推进镜头,强调角色惊讶
  ↓ 辩论 ↓
导演裁决: 采用推进镜头,因为这场戏的重点是情绪反应
```

### 实验数据 (论文结果)

| 方法 | 动作准确率 | 剧情连贯 | 角色匹配 | 镜头合适 | 平均分 |
|------|-----------|---------|---------|---------|--------|
| 单Agent CoT | 0.68 | 1.60 | 3.84 | 1.67 | 2.63 |
| 单Agent o1 | 0.80 | 2.73 | 3.60 | 2.86 | 3.30 |
| FilmAgent Solo | 0.80 | 1.87 | 4.20 | 2.07 | 3.04 |
| **FilmAgent Group** | **0.88** | **3.53** | **4.44** | **3.53** | **3.98** |

**关键发现:**
- 多Agent协作(GPT-4o) > 单Agent(o1) 
- 协作能显著提升剧情连贯性和镜头选择
- 减少"幻觉"(不存在的动作、不合理的镜头)

## 🎥 9种镜头类型 (FilmAgent定义)

### 3种静态镜头
| 类型 | 英文 | 画面范围 | 用途 |
|------|------|----------|------|
| **特写** | Close-up (CU) | 面部+领口 | 身份识别、情绪 |
| **中景** | Medium Shot (MS) | 腰部以上 | 肢体语言、动作 |
| **远景** | Long Shot (LS) | 全身+环境 | 位置、环境关系 |

### 6种动态镜头
| 类型 | 英文 | 运动方式 | 用途 |
|------|------|----------|------|
| **平移** | Pan Shot | 水平转动 | 跟随、展示宽度 |
| **推进** | Zoom Shot | 向前推进 | 强调、聚焦 |
| **弧形** | Arc Shot | 绕主体旋转 | 全方位展示 |
| **跟拍** | Tracking Shot | 跟随移动 | 运动场景 |
| **摇镜** | Tilt Shot | 垂直转动 | 展示高度 |
| **环绕** | Orbit Shot | 360度环绕 | 炫技、全景 |

### 镜头选择决策树

```
要表达什么?
    │
    ├─ 身份/情绪 → 特写 (CU)
    │
    ├─ 肢体语言/动作 → 中景 (MS)
    │
    ├─ 位置/环境 → 远景 (LS)
    │
    ├─ 强调/惊讶 → 推进 (Zoom)
    │
    ├─ 跟随移动 → 跟拍 (Tracking)
    │
    └─ 全方位展示 → 弧形/环绕 (Arc/Orbit)
```

### 镜头组合原则

**1. 景别变化要有节奏**
- ❌ 连续5个中景 (单调)
- ✅ 远景→中景→特写→中景 (有变化)

**2. 情绪升级配合景别收紧**
- 平静对话: 中景
- 冲突升级: 中近景
- 情绪爆发: 特写
- 关键时刻: 极特写

**3. 180度法则**
- 两人对话时,摄像机保持在同一侧
- 避免观众方向感混乱

## 📝 21种角色动作 (FilmAgent定义)

### 基础动作
- 站立 (Standing)
- 坐下 (Sitting down)
- 行走 (Walking)
- 转身 (Turning)

### 情绪动作
- 开心跳跃 (Joyful jumping)
- 烦躁摇头 (Annoyed head-shaking)
- 惊讶后退 (Surprised stepping back)
- 悲伤低头 (Sad head down)
- 愤怒挥手 (Angry waving)

### 交互动作
- 握手 (Handshaking)
- 拥抱 (Hugging)
- 指向 (Pointing)
- 鼓掌 (Clapping)

**应用:** 在提示词中明确指定角色动作,提高生成准确性

## 🔄 FilmAgent vs Sora/Seedance 对比

| 维度 | FilmAgent | Sora/Seedance |
|------|-----------|---------------|
| **适应性** | 需要预设3D空间 | 高,任意场景 |
| **一致性** | 高,角色稳定 | 低,容易变形 |
| **叙事能力** | 强,有完整剧本 | 弱,难以讲故事 |
| **物理合规** | 高,基于3D引擎 | 低,有奇怪伪影 |
| **创意自由** | 受限于预设 | 高,无限可能 |

**结论:** 两者互补
- FilmAgent思路: 适合需要一致性和叙事的内容
- Seedance: 适合创意探索和多样化场景
- **我们的方案**: 用FilmAgent的协作思路 + Seedance的生成能力

## ✏️ 分镜快速验证 (Storyboarder)

### 核心理念
> "你是来画故事的,不是来画蒙娜丽莎的"

### 分镜要素
每格分镜应包含:
- **画面**: 简单草图即可
- **镜头类型**: 远/中/近/特写
- **运镜**: 推/拉/摇/移/固定
- **时长**: 大约几秒
- **对白/音效**: 关键声音
- **备注**: 特殊要求

## 🎯 我的工作方式 (基于FilmAgent)

### 协作策略应用

**1. 给镜头方案时 (Debate-Judge)**
```
方案A: 中景,捕捉肢体语言
- 优点: ...
- 缺点: ...

方案B: 推进镜头,强调情绪
- 优点: ...
- 缺点: ...

我的建议: X,因为...
海哥你来裁决?
```

**2. 审核剧本时 (Critique-Correct-Verify)**
```
第一轮: 我写初稿
第二轮: 我自己审核,找问题
第三轮: 修正后给海哥确认
```

# 多人打斗空间位置正确性实操指南

> 2026-06-25 整合建立。来源：SELF_CHECK.md 空间逻辑 + Wolfcrow 3+人轴线 + USER.md 两人站位 + 传统电影Blocking + Shot/Reverse Shot焦点组合 + 知乎Seedance一镜一主体硬限。

---

## 一、核心原则（落笔前必读）

### 1.1 物理坐标替代世界坐标

模型只认画面内坐标，不懂"北边""秦牧方向"等世界坐标。

| ❌ 世界坐标 | ✅ 物理坐标 |
|------------|-----------|
| 秦牧左手边 | 画左 |
| 怪从背后袭来 | 怪从画深处冲向画前 |
| 围住秦牧 | 三怪分列画左/画右/画深处 |

### 1.2 模型硬限：一镜一主体一核心动作

**Seedance生成能力决定：** 一个15秒片段内，每镜只能聚焦1个主体+1个核心动作。多人多动作必须拆镜分段。

- ✅ 镜头N：秦牧（主体）横扫画左怪（动作）→ 画面里只重点写这两人
- ❌ 镜头N：秦牧砍画左怪+画右怪同时扑来+画深处怪也在出招 → 三件事一个镜头=崩

### 1.3 一人一段位，不混写

- **秦牧总是画中心**（包围圈主位）
- **每镜只写一个对手**作为画面第二人
- 其余对手在画面外的，只在【站位.剧情】里交代存在，不在该镜内容里写

## 二、写前锁定：多人空间布局表（模版）

每段提示词落笔前，先填这张表：

```
【空间布局锁定】

场景朝向基准：______________（如：画面右侧=正午阳光来源）

人物：
- 秦牧：（位置）画______（朝向）面朝______
- 对手A：（位置）画______（朝向）面朝______（编号/特征）______
- 对手B：（位置）画______（朝向）面朝______（编号/特征）______
- 对手C：（位置）画______（朝向）面朝______（编号/特征）______

轴线定义：秦牧与______（主对手）之间的连线为轴线

摄影机侧：轴线______侧（全段所有镜头必须在此侧180°内）

主镜建立：镜头___（用全景/中景一次交代所有人位置关系）
```

**填表示例（秦牧vs三怪三角包围）：**

```
场景朝向基准：画面右侧=正午阳光来源

人物：
- 秦牧：画中心，面朝画右略偏前
- 对手A：画左前方，面朝画右（向秦牧），人形怪A·体型魁梧
- 对手B：画右前方，面朝画左（向秦牧），人形怪A·爪最长
- 对手C：画深处正面，面朝画前（向秦牧），人形怪A·面甲破损

轴线定义：秦牧与对手B（画右前方主威胁）之间的连线为轴线

摄影机侧：轴线画前侧（机位在秦牧前方偏左，始终不动）

主镜建立：镜头1（0-3秒）WS俯拍，四方位置一次交代
```

## 三、多人（3+）180度轴线：整体单元法

> 来源：Wolfcrow "180-degree rule with three and more characters"

### 3.1 双人的轴线规则（复习）

双人时，轴线=两人之间的视线连线。机位在轴线一侧，两人在画面中的左右关系永远不变。

```
秦牧（画左）←→ 对手（画右）
     │
   [轴线]
     │
  摄影机始终在画前侧
```

### 3.2 三人及以上的轴线处理

**把多人组当成一个"整体单元"**，从这个整体单元的外缘画一条虚拟轴线。不是在每两人之间画线——那是混乱的根源。

```
        对手C（画深处）
       /           \
      /   包围圈     \
     /（整体单元）    \
对手A（画左） ——— 对手B（画右）
     |        |
     |   秦牧  |
     |  （画中心）|
     |           |
   [轴线：包围圈切面]
     |
  摄影机在画前侧
```

**关键操作：**
1. 先定义包围圈/多人组的"整体单元边界"
2. 在这个单元外侧画一条轴线
3. 所有镜头的机位都在轴线同侧
4. 切入单人镜头时，人物在画面中的左右归属不变

### 3.3 环形包围的轴线简化

环形包围 = 把包围圈当成一个"圆"，在圆的外侧画切线作为轴线。

**简化法：** 选一个"主视角方向"（通常是秦牧面朝的方向），所有镜头从同侧拍。

```
环形包围（俯视图）：

         对手C
        /     \
       /  秦牧  \
      /    ★     \
   对手A ——— 对手B

主视角方向 → 秦牧面朝画右

摄影机侧：始终在秦牧前方偏左（画前侧）
→ 秦牧在画中偏左，面对画右
→ 对手A永远在画左深处
→ 对手B永远在画右
→ 对手C永远在画上方/远处
```

## 四、两人同框站位标准化写法

> 来源：USER.md §四B

画面中有两人及以上时，【人物】字段和镜头内容按以下标准写：

### 4.1 人物字段格式

```
角色名（画左/右/中+位置+朝向+状态）
```

**示例：**
```
秦牧（画中心站立，右手持刀，面朝画右前方）
对手A（画左前方，面朝秦牧方向，双爪蓄势，暗色特效从爪尖翻涌）
```

### 4.2 镜头内容中的站位描写

每镜内容中嵌入位置关系：

```
✅ 秦牧侧身闪过画左对手A的爪击——借旋转惯性贴地滑向画右对手B
✅ 秦牧（画中心）刀尖前刺贯穿画深处对手C胸腹
❌ 秦牧闪过对手攻击后砍向另一个 → 没写空间位置，模型会乱
```

## 五、主镜→分镜→重立主镜 工作流

> 来源：传统电影Blocking

### 5.1 三步工作流

```
主镜（建立空间）
    ↓
分镜1-N（逐个焦点覆盖）
    ↓
重立主镜（重置空间认知）
```

### 5.2 主镜——建立空间（必须第1镜）

**全段第一个镜头永远是全景或中景**，一次性交代：

- 谁在哪里（所有人物的画左/右/中/深位置）
- 空间关系（间距、包围还是对峙）
- 光线方向（用光影暗示空间朝向）

```
✅ 镜头1（0-3秒）：WS全景俯拍
   秦牧站画中心，三只怪分列画左/画右/画深处形成三角包围。
   暖金丁达尔光束从画右上斜射入画。
```

### 5.3 分镜——逐焦点覆盖（交替拍）

每镜聚焦1个秦牧→对手交互对。但相邻两镜不能重复同一个对手组合。

**焦点轮换示例（秦牧vs三怪5镜）：**

| 镜头 | 焦点 | 景别 | 其他对手在画面中的状态 |
|------|------|------|----------------------|
| 镜1（主镜） | 四人都可见 | WS俯拍 | 全在画面内，建立空间 |
| 镜2 | 秦牧→对手A | MS跟拍 | B和C在画外，只在【站位】提及 |
| 镜3 | 秦牧→对手B | MS跟拍 | A已倒地（画外），C在画外 |
| 镜4 | 秦牧→对手C | CU特写 | B已倒地（画外），C作为焦点 |
| 镜5（重立主镜） | 秦牧+全部倒地 | WS仰拍 | 三怪残骸放射状分布 |

### 5.4 重立主镜——重置空间（段尾或关键转折点）

当所有对手都被打倒、或秦牧位移跨过包围圈中线后，必须用一镜全景重新建立空间。

```
✅ 镜头5（12-14秒）：WS大全景 Low Angle仰拍缓拉
   秦牧独自站立，三怪残骸放射状分布在脚边 → 观众重新确认"现在谁在哪"
```

## 六、焦点组合法（Shot/Reverse Shot的多人扩展）

> 来源：传统Shot/Reverse Shot的多人扩展

### 6.1 从双人到多人：先选"焦点组合"

Shot/Reverse Shot（正反打）的本意是：A镜拍秦牧→B镜拍对手→A镜再回秦牧。

**多人时需要先回答：这一镜的"焦点组合"是谁+谁？**

```
焦点组合 = 镜头内最核心的两个（或两个以内）人物

例：秦牧vs对手A → 焦点组合 = {秦牧, 对手A}
    → 镜头拍秦牧和对手A，其余对手不入画
```

### 6.2 焦点组合的轮换规则

- 每镜只选1对焦点组合
- 相邻两镜不能重复同一对
- 焦点组合换人时，用过渡信息交代（如：秦牧砍倒A→顺势转身面向B）

```
镜2：焦点{秦牧, 对手A} → 秦牧一刀斩倒A
镜3：焦点{秦牧, 对手B} → 秦牧借旋转惯性滑向B
    ↑ 过渡信息："秦牧从A身侧穿透而过，顺势转身面向B" 连接了两个焦点
```

### 6.3 多人正反打的简化

三人及以上不逐人正反打——那会导致镜头碎片化。

**替代方案：中景跟拍一人多杀 + 穿插对手POV低角度仰拍作为"反打"**

```
✅ 秦牧中景跟拍横扫（正打）
✅ 倒地对手低角度仰拍看秦牧（反打——从对手视角）
❌ 秦牧→对手A→秦牧→对手B→秦牧→对手C（碎成渣）
```

## 七、逐镜位置追踪检查清单

每段分镜写完，逐镜过这5条：

```
[ ] 镜1：是否有主镜建立空间？（全景或中景，所有人位置一次交代）
[ ] 每镜：焦点组合是哪一对？其余人在画外还是画内？
[ ] 每镜：秦牧的画左/右归属是否全程一致？（秦牧画中心 → 永不变）
[ ] 每镜：对手入画时，左右位置是否与主镜建立的布局一致？
[ ] 镜末：是否用全景重立了空间？（尤其多人被击倒/位移后）
```

## 八、常见空间错误与修正

| 错误 | 表现 | 修正 |
|------|------|------|
| 越轴 | A镜秦牧画左看右，B镜对手也在画左看右 → 两人看向同一方向 | 锁定摄影机侧，全段不换 |
| 对手漂移 | 镜2对手在画右，镜3他突然在画左（没写他移动） | 镜间保持对手左右归属不变，除非写了移动动作 |
| 失焦多人 | 一镜写了秦牧+三个对手同时出招 → 四个动作塞一镜 | 拆镜，每镜只写秦牧+一个对手 |
| 缺主镜 | 第一个镜头直接开始打 → 观众不知道空间关系 | 镜1永远是全景/中景建立空间 |
| 忘重立主镜 | 打完所有对手直接收尾 → 观众忘了"谁倒在哪" | 段尾扫一镜全景，交代残局空间 |
| 正面冲突 | 写秦牧"正面面对镜头"但他应该面对对手 | "正面"必须标注"正面面对对手方向"或"侧脸对镜头" |

## 九、快速决策流（写分镜前30秒走完）

```
1. 多少人？→ 2人直接走双人流程 / 3+人往下走
2. 空间形态？→ 对峙线/三角包围/环形包围/混战
3. 填空间布局表（第二章模版）
4. 选定主视角方向 → 定摄影机侧
5. 编号所有对手 → 分焦点组合 → 排轮换顺序
6. 镜1=主镜 → 中间=逐焦点分镜 → 末镜=重立主镜
7. 写完逐镜过位置追踪检查（第七章5条）
```

# AI剧组编剧系统

基于 story-systems-template 的多AI协作编剧方法。

## 核心理念

把AI从"一个万能实习生"变成"一个专业剧组",11个角色各司其职。

---

## 👥 11个AI角色

### 导演组 (管故事)
| 角色 | 职责 | 权限 |
|------|------|------|
| **Story Architect** | 把控整体结构 | ⚠️ 一票否决权 |
| **Story Analyst** | 逐场分析找问题 | 建议权 |

### 编剧组 (写内容)
| 角色 | 职责 | 专长 |
|------|------|------|
| **Dialogue Writer** | 写对白 | 让每个角色说话风格不同 |
| **Scene Writer** | 写场景 | 动作、画面、氛围描述 |
| **Scene Annotator** | 做笔记 | 整理素材、标注要点 |

### 质检组 (查错误)
| 角色 | 职责 | 权限 |
|------|------|------|
| **Continuity Editor** | 查连续性 | 抓穿帮、时间线错误 |
| **Script Supervisor** | 查格式 | ⚠️ 导出前必须签字 |
| **Standards Reviewer** | 查规范 | 确保符合行业标准 |

### 后勤组 (支持)
| 角色 | 职责 |
|------|------|
| **Research Specialist** | 查资料、事实核查 |
| **Production Coordinator** | 导出PDF/视频脚本 |
| **Session Manager** | 协调工作流程 |

## 🎬 故事结构模板

### 三幕结构 (最常用)
```
第一幕 (25%) - 建置
├── 开场 - 展示日常世界
├── 激励事件 - 打破平衡
└── 第一转折点 - 进入新世界

第二幕 (50%) - 对抗
├── 上升动作 - 主角尝试解决
├── 中点 - 重大转折/假胜利
├── 下降动作 - 困难加剧
└── 第二转折点 - 最低谷

第三幕 (25%) - 解决
├── 高潮 - 最终对决
└── 结局 - 新的平衡
```

### Save the Cat! 15节拍
```
1. 开场画面 (1%)
2. 主题陈述 (5%)
3. 铺垫 (1-10%)
4. 催化剂 (10%)
5. 辩论 (10-20%)
6. 进入第二幕 (20%)
7. B故事 (22%)
8. 玩乐时光 (20-50%)
9. 中点 (50%)
10. 坏人逼近 (50-75%)
11. 一无所有 (75%)
12. 灵魂黑夜 (75-80%)
13. 进入第三幕 (80%)
14. 结局 (80-99%)
15. 终场画面 (99-100%)
```

### 英雄之旅 12阶段
```
普通世界 → 冒险召唤 → 拒绝召唤 → 遇见导师
→ 跨越门槛 → 考验/盟友/敌人 → 接近洞穴
→ 磨难 → 奖赏 → 返回之路 → 复活 → 带着万灵丹归来
```

## 📝 场景类型模板

### 对峙场景
```
场景目标: [角色A想要什么]
障碍: [角色B为什么阻止]
赌注: [失败会怎样]

节奏:
1. 试探 - 双方摸底
2. 升级 - 冲突加剧
3. 转折 - 意外发生
4. 结果 - 一方获胜/僵局
```

### 发现场景
```
铺垫: [观众知道什么,角色不知道什么]
发现时刻: [角色如何发现真相]
反应: [情感冲击]
后果: [这改变了什么]
```

### 蒙太奇场景
```
目的: [展示时间流逝/技能提升/关系发展]
画面序列:
- 画面1: [描述] + [情感基调]
- 画面2: [描述] + [进展]
- 画面3: [描述] + [高潮]
音乐建议: [节奏/情绪]
```

## 🗣️ 对白写作技巧

### 角色声音区分
```
角色A (教授): 
- 用词: 学术、精确
- 句式: 长句、从句多
- 口头禅: "从理论上讲..."

角色B (街头混混):
- 用词: 俚语、简短
- 句式: 断句、省略
- 口头禅: "你懂的"
```

### 潜台词技巧
```
表面说: "天气真好"
实际意思: "我不想谈这个话题"

技巧:
- 说一半留一半
- 用行动代替语言
- 答非所问
```

### 冲突对白模板
```
A: [表面诉求]
B: [拒绝/反驳]
A: [升级/揭露真实动机]
B: [防御/反击]
A: [最后通牒]
B: [选择]
```

## ✅ 质检清单

### 连续性检查
- [ ] 时间线是否连贯
- [ ] 角色服装/道具是否一致
- [ ] 角色知道的信息是否合理
- [ ] 地点转换是否有逻辑

### 角色检查
- [ ] 每个角色有独特说话方式吗
- [ ] 角色动机清晰吗
- [ ] 角色有成长弧线吗
- [ ] 配角有存在必要吗

### 结构检查
- [ ] 开场5分钟能抓住观众吗
- [ ] 中点有足够转折吗
- [ ] 高潮是否是全片最强时刻
- [ ] 结局是否呼应开场

## 🔗 与Seedance的衔接

### 工作流程
```
1. 用本系统写剧本/分镜
2. 提取每场戏的视觉描述
3. 转换为Seedance提示词
4. 生成视频
5. 剪辑组合
```

### 剧本→提示词转换示例

**剧本场景:**
```
INT. 咖啡馆 - 日

阳光透过落地窗洒进来。SARAH (30s) 独自坐在角落,
手指无意识地转动咖啡杯。她看向窗外,眼神空洞。

门铃响起。她抬头,表情瞬间紧绷。
```

**转换为Seedance提示词:**
```
咖啡馆内景,温暖的午后阳光透过落地窗,
一位30岁左右的女性独坐角落,
手指轻轻转动咖啡杯,眼神望向窗外,
表情从空洞转为紧张,
镜头缓慢推进特写她的表情变化,
电影质感,柔和光线,4K画质
```

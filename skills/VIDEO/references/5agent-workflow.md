# 视频制作通用技能（完全匹配用户定义的5Agent分工）
## 前置规则
每次生成新视频项目时，**必须优先读取本技能文档**，严格遵循角色分工流程执行，不得私自简化角色。
---
## 固定Agent角色配置（100%匹配用户定义）
| Agent ID | 名称 | 固定使用模型 | 专属职责 |
| --- | --- | --- | --- |
| video-writer | 编剧 | deepseek/deepseek-reasoner | 根据制作要求生成文生图提示词、剧情内容、镜头内容 |
| video-modeler | 建模师 | deepseek/deepseek-reasoner | 根据提示词生成参考图片/场景/人物素材 |
| video-director | 导演 | deepseek/deepseek-reasoner | 根据剧情和图片生成运镜脚本、分镜、节奏要求 |
| video-cameraman | 摄影师 | deepseek/deepseek-reasoner | 调用Seedance/Seedream API生成视频、处理生成失败重试 |
| video-scribe | 场记 | MiniMax-M2.5 | 记录更新剧本信息、用户喜好、制作流程工艺、沉淀优化方案 |
---
## 全流程执行标准
### 阶段0：项目启动与剧本概况确认（仅video-scribe参与）
**必须在开始创作新的视频项目之前完成，所有内容必须得到主人确认**

1. video-scribe主动与主人确认整体剧情概况，或接收主人提供的剧本
2. 场记必须记录并确认以下关键信息，创建《项目整体说明》文档：
   - **故事背景**：年代、国家、社会环境、剧情风格、艺术风格等
   - **剧情主线**：剧情类型（感情戏、喜剧、悲剧、恐怖、仙侠等）、章节长短、主要角色关系
   - **视频规格**：视频比例、分辨率、是否需要同时输出声音、目标平台要求
   - **特殊要求**：物理常识约束、人文习俗限制、生活常识要求
3. 将《项目整体说明》发送给主人确认，主人确认无误后才能进入阶段1
4. 创建视频项目文件夹，所有后续文档和素材都存储在该项目中

### 阶段1：需求归档与项目初始化（仅video-scribe参与）
1. video-scribe基于确认的《项目整体说明》，整理《项目需求基准文档》
2. 同步文档给所有其他4个Agent，确保所有角色信息一致
3. 强调项目整体说明中的关键约束条件，特别是物理常识、人文习俗、生活常识要求
### 阶段2：内容创作（video-writer独立执行）
**必须严格遵循项目整体说明，不得自作主张添加不符合剧情和物理常识的内容**

1. 基于《项目整体说明》和《项目需求基准文档》，生成方便建模师、导演、摄影师使用的剧情分析
2. 输出《剧情分析文档》包含：
   - 完整剧情分镜文案
   - 文生图/文生视频提示词
   - 角色关系分析
   - 关键场景描述
3. 将《剧情分析文档》提交给主人确认，主人确认无误后才能进入阶段3
4. 场记记录主人对剧情分析的反馈意见，如有修改要求则更新到《项目整体说明》中
### 阶段3：并行素材与脚本生成（video-modeler和video-director并行执行）
**用户确认剧情分析后，同时启动建模师和导演工作**

#### 3A：建模师工作（video-modeler）
1. 基于编剧输出的文生图提示词，调用火山引擎API，使用模型 **doubao-seedream-5-0-260128** 生成统一风格的人物、场景、道具的三视图参考图
2. 输出《视觉参考素材包》包含：
   - 主要角色三视图（正面、侧面、背面）
   - 关键场景多角度视图
   - 重要道具细节图
3. 提交给主人确认

#### 3B：导演工作（video-director）
1. 基于剧情分析和项目整体说明，生成符合剧情的运镜脚本
2. 输出《视频运镜脚本》包含：
   - 时间切片（每个镜头的时间点）
   - 画面描述（每个镜头的具体内容）
   - 运镜方式（推、拉、摇、移、跟等）
   - 转场方式（切、淡入淡出、划变等）
   - 节奏要求（快慢、紧张舒缓）
3. 提交给主人确认

#### 3C：用户综合确认
1. 主人需要同时确认建模师的《视觉参考素材包》和导演的《视频运镜脚本》
2. 两者都确认无误后才能进入阶段4
3. 场记记录主人对素材和脚本的反馈意见，特别是：
   - 不符合物理常识的内容
   - 不符合人文习俗、生活常识的内容  
   - 不符合剧情逻辑的内容
4. 根据主人意见更新《项目整体说明》，确保后续制作不犯同样错误
### 阶段4：分片段视频生成与确认（video-cameraman独立执行）
**每个视频片段都必须得到主人确认，严格按照项目整体说明执行**

1. 在开始生成任何视频前，摄影师必须重新阅读《项目整体说明》，确保理解所有要求
2. 基于编剧的提示词+导演的运镜脚本+建模师的参考图，按镜头顺序生成视频片段：
   - 每个镜头单独生成一个视频文件
   - 每个视频生成后立即提交给主人确认
   - 主人确认当前片段无误后才能生成下一个片段
3. **场景一致性强制要求**：
   - 如果多个镜头在同一场景内连续，每个视频的第一帧必须使用上一个视频的最后一帧作为首帧参考图
   - 必须保持场景内所有物品的布局不变，人物、服装、场景、色调、尺寸必须与上一个视频完全一致
   - 严格禁止出现“幻觉”（物品突然出现/消失、人物服装突变、场景布局变化等）
4. 视频生成技术规范：
   - 调用火山引擎API，使用模型 **doubao-seedance-1-5-pro-251215** 生成视频
   - 所有字幕、文字内容直接包含在提示词中由视频模型生成，禁止使用本地ffmpeg硬编码文字
   - 生成失败自动优化提示词重试，最多3次
5. 场记记录每个片段的生成结果和主人反馈：
   - 如果主人不满意，必须精确记录不满意的具体点
   - 将用户指出的错误点、修正意见提炼出来，更新到《项目整体说明》
   - 这些意见是“总编导的指导意见”，必须高度重视并应用于后续制作
### 阶段5：最终质检与知识沉淀（video-scribe独立执行）
**全程记录，持续更新项目整体说明**

1. 检查所有视频片段是否符合《项目整体说明》的所有要求，特别关注：
   - 物理常识符合性（重力、运动规律、材质表现等）
   - 人文习俗符合性（服装、礼仪、建筑风格等）
   - 生活常识符合性（日常行为逻辑、物品使用方式等）
   - 场景布局一致性（连续镜头中的物品位置、人物状态等）
2. 不合格视频打回对应角色重制，并记录具体不合格原因
3. **核心职责：对话记录与知识提炼**
   - 记录与主人的每次对话，特别是：
     - 主人指出的错误点
     - 主人不满意的结果及原因
     - 主人提出的修正意见
   - 将这些意见提炼成明确的指导原则，更新到《项目整体说明》
   - 确保后续制作不再犯同样错误
4. 将项目经验沉淀到通用工艺库，优化未来视频制作流程
5. 最终合格视频整合交付给主人，并提供完整的制作过程记录

## 通用规范
### 1. 内容创作基本原则
- **严格遵循剧本**：不得自作主张添加不符合剧情和物理常识的内容
- **符合现实逻辑**：所有内容必须符合物理常识、人文习俗、生活常识
- **尊重用户意见**：用户反馈是“总编导的指导意见”，必须高度重视并立即应用

### 2. 场景一致性强制要求
- **连续镜头处理**：同一场景内的连续镜头，每个视频的第一帧必须使用上一个视频的最后一帧作为首帧参考图
- **布局保持不变**：必须保持场景内所有物品的布局不变，禁止出现“幻觉”（物品突然出现/消失、人物服装突变等）
- **视觉元素一致**：人物、服装、场景、色调、尺寸必须与前后镜头完全一致

### 3. 视频技术规范
- **输出尺寸**：默认统一为 **1280×720（720p 16:9）**，特殊要求除外
- **多段视频衔接**：必须用`ffmpeg`提取上一段视频最后一帧作为首帧输入，保证场景/人物/色调100%一致
- **过审规则**：敏感内容自动替换，暴力/血腥内容用意象表述，保留核心需求的前提下过审
- **文字内容**：所有字幕/文字内容直接写入提示词，由doubao-seedance视频模型直接生成，禁止使用本地ffmpeg硬编码文字

### 4. 用户确认流程
- **每个环节必须确认**：阶段2-5的每个输出都必须得到主人明确确认后才能进入下一步
- **场记全程记录**：场记必须记录每次对话，特别是用户不满意的点和修正意见
- **实时更新文档**：用户意见必须实时提炼并更新到《项目整体说明》，指导后续制作

# 视频制作经验总结 (2026-02-25 实战提炼)

基于贾维斯出场视频制作过程的经验总结。

---

## ⚠️ 常见问题与避坑指南

### 1. 图片与视频比例匹配
- ❌ 错误: 3:1宽图 → 视频拒绝(宽高比需在0.4-2.5之间)
- ✅ 正确: 16:9 (2848x1600) → 完美适配视频
- **教训**: 图生视频前,先确认图片比例

### 2. 模型ID要准确
- ❌ `doubao-seedream-5.0-lite` (文档里的名字)
- ✅ `doubao-seedream-5-0-260128` (实际调用ID)
- **教训**: 从控制台代码示例复制,不要猜

### 3. 多人场景的人物高度比例 ⭐重要
- **问题**: 多人同框时,身高比例不协调(主角太高或太矮)
- **解决**: 提示词中明确写入人物高度比例
- **示例**:
  ```
  主角身高约175cm,与周围行人身高比例协调一致
  人群中主角不突兀,自然融入环境
  前景人物与背景人物保持透视比例正确
  ```
- **检查点**: 生成后检查人物大小是否合理

### 4. 视频时长限制
- Seedance 1.5 Pro: 5秒或10秒
- **策略**: 复杂运镜用5秒,避免失败

### 5. 图生视频的平衡
- 提示词太多 → 可能偏离参考图
- 提示词太少 → 动态不够
- **平衡点**: 明确运镜+关键动态,其他依赖参考图

## ✅ 成功要点

### 1. 场景一致性控制
**方法**: 图生图/图生视频 + 参考图输入

**关键词**: 
```
保持参考图中XXX完全一致
保持参考图中的场景、建筑、色调、霓虹风格高度完全一致
```

### 2. 提示词结构化
借鉴官方示例的分块结构效果更好:
```
【场景】具体环境描述
【人物/动态】角色和动作(含身高比例)
【运镜】镜头运动方式
【风格】导演风格参考
【色彩与光线】色调+光效
【质感】渲染方式+分辨率
```

### 3. 运镜描述要具体
- ❌ 模糊: "镜头移动"
- ✅ 清晰: "镜头快速斜向上拉升爆发,如火箭般冲天,掠过建筑侧面"
- **关键**: 动作+速度+路径+效果

### 4. 导演风格融入
**王家卫风格关键词**:
- 抽帧跳跃感
- 霓虹色彩流转(红蓝绿粉紫)
- 高对比度光影
- 胶片颗粒质感
- 浅景深背景虚化

### 5. 细节丰富化
- 街道场景加入: 行人+雨伞+AR眼镜+小贩+飞行器
- 环境动态: 霓虹闪烁+数据流+云层飘动+雨后反射
- **效果**: 画面更有生命力

## 🎯 提示词技巧

### 技巧1: 三段式描述
```
1. 开场画面 (1-2秒)
2. 运镜过程 (核心动作)
3. 结尾定格 (最后1-2秒)
```

### 技巧2: 运镜+场景动态分开
```
【运镜】镜头如何移动
【场景动态】场景内元素如何变化(霓虹闪烁、人群移动)
```

### 技巧3: 镜头语言术语
| 中文 | 英文 | 效果 |
|------|------|------|
| 升降 | Crane up/down | 史诗感、揭示 |
| 推进 | Dolly in / Push in | 聚焦、强调 |
| 拉远 | Pull back / Dolly out | 揭示全景 |
| 环绕 | Orbit | 360度展示 |
| 一镜到底 | One-take | 流畅、真实感 |
| FPV视角 | FPV drone | 穿越感、沉浸 |

### 技巧4: 视觉冲击关键词
- 史诗感: epic scale, sweeping
- 震撼: stunning, breathtaking
- 流畅: smooth, fluid
- 加速: accelerating, rushing
- 电影级: cinematic, film-grade

### 技巧5: 人物比例关键词 ⭐
```
人物高度比例协调一致
主角与周围人物身高自然
透视比例正确
人物在画面中的相对大小合理
```

## 🔄 工作流总结

```
1. 构思故事/场景
   ↓
2. 生成16:9参考图 (Seedream 5.0)
   - 尺寸: 2848x1600 (16:9适配视频)
   - 结构化提示词
   - 多人场景注意身高比例
   ↓
3. 图生视频 (Seedance 1.5 Pro)
   - 引用参考图
   - 明确运镜(动作+速度+路径)
   - 场景一致性要求
   ↓
4. 检查与迭代
   - 检查人物比例
   - 检查运镜是否符合预期
   - 需要则调整提示词重新生成
   ↓
5. 多段拼接(如需要)
   - 保持场景一致
   - 运镜衔接自然
```

### 6. 人物素材审核问题处理 ⭐重要
- **问题**: 使用真实人物素材时，即梦可能因肖像权或内容审核拒绝生成
- **解决方案**: 使用PS等工具处理人物脸部特征
  - **技巧1**: 缩小脸部特征（眼睛、鼻子、嘴巴等）
  - **技巧2**: 轻微调整面部轮廓
  - **技巧3**: 改变发型或发色
  - **技巧4**: 添加眼镜、帽子等配饰
- **注意事项**:
  - 脸部变形是必然的副作用，需在可接受范围内
  - 保持人物基本辨识度与角色特征
  - 处理后的人物应更像"艺术创作"而非真实肖像
- **应用场景**:
  - 短剧制作中使用真实演员素材
  - 商业广告中避免肖像权问题
  - 需要特定外貌但无法通过纯文本描述实现时

### 7. 审核规避策略
1. **预处理素材**: 在输入前对人物图片进行艺术化处理
2. **特征弱化**: 减少过于明显的个人特征
3. **风格化**: 添加滤镜或艺术效果
4. **组合处理**: 使用多个参考人物的混合特征

### 8. 裸体场景审核技巧 ⭐重要 (2026-04-23 新增)
- **问题**: 即梦对裸体内容审核严格，直接描述"裸体"、"裸露上半身"等会被拒绝
- **解决方案**: 使用艺术化、专业化的描述替代
  - **技巧1**: "展示人物上半身肌肉" - 用于男性角色裸露上半身场景
  - **技巧2**: "展示人物身体线条" - 用于健身、艺术人体场景
  - **技巧3**: "展示人物背部/肩部肌肉结构" - 用于特定部位展示
  - **技巧4**: "人物身着轻薄贴身衣物" - 用于若隐若现效果
- **注意事项**:
  - 避免直接使用"裸体"、"裸露"等敏感词汇
  - 使用专业健身、艺术、医学等领域的描述词汇
  - 结合场景上下文，如"健身训练中展示肌肉线条"
  - 保持艺术性和专业性的描述角度
- **应用场景**:
  - 健身/运动类视频中展示肌肉
  - 艺术人体摄影/绘画参考
  - 医疗/解剖学教学视频
  - 古装/武侠剧中受伤包扎场景

**示例对比**:
- ❌ 错误: "男人裸体躺在床上"
- ✅ 正确: "男人展示上半身肌肉线条，躺在床上休息，电影级光影效果"
- ❌ 错误: "女人裸露上半身"
- ✅ 正确: "女人展示身体线条，身着轻薄贴身衣物，艺术人体摄影风格"

## 🚫 物理崩塌 & 剧情脱节检查清单 (2026-04-22 新增)

### 核心原则
**写提示词前，必须逐条检查以下规则，确保每一帧都符合真实物理规律和剧情逻辑。**

### 一、物理规律类检查项

#### 1. 动作逻辑连贯性
- [ ] 拧瓶盖→拔瓶塞→开盖→上药（不能跳步骤）
- [ ] 拿东西→手部要有明确的抓取动作，不能凭空出现在手里
- [ ] 放东西→要有放下/松手的动作序列
- [ ] 物体交互→手和物体接触位置要合理
- [ ] 穿衣/脱衣→要有明确的穿脱过程
- [ ] 开关门→手触门把→旋转→推/拉→门开/关

#### 2. 物体状态一致性
- [ ] 玻璃碎了就是碎了，不能再恢复成完整花瓶
- [ ] 水倒了就是倒了，杯子不能自己立起来
- [ ] 破碎/撕裂/变形→不可逆状态变化
- [ ] 消失/出现→必须有明确动作解释
- [ ] 液体泼洒后衣服上的湿痕要持续
- [ ] 被撕碎的文件不能恢复完整

#### 3. 空间/透视逻辑
- [ ] 人物在画面中的相对位置保持一致
- [ ] 近大远小透视正确
- [ ] 遮挡关系合理（人走后背景恢复，不能留半个人影）
- [ ] 影子方向、倒影位置符合光源
- [ ] 多人同框时身高比例协调
- [ ] 前后景人物大小符合透视关系

#### 4. 重力与运动
- [ ] 物体自由落体速度合理（不是慢慢飘）
- [ ] 被打倒后不会立刻站起（需要合理时间）
- [ ] 跑步时衣摆/头发摆动方向符合运动方向
- [ ] 水的流动、烟的飘散符合物理
- [ ] 跳起→落下→有缓冲动作

### 二、剧情/情绪类检查项

#### 5. 表情-语气-剧情三角一致
- [ ] 愤怒的语气 → 愤怒的表情 → 愤怒的原因（前情铺垫）
- [ ] 哭泣 → 泪痕 → 红眼眶 → 哽咽声音
- [ ] 惊喜 → 眼睛睁大 → 嘴角上扬 → 呼吸加快
- [ ] 恐惧 → 瞳孔收缩 → 身体后倾/颤抖
- [ ] 台词的情绪和面部表情必须匹配

#### 6. 时间线/因果关系
- [ ] 前一秒还重伤躺地，不能下一秒活蹦乱跳
- [ ] 前一集刚吵架决裂，不能这一集开场就卿卿我我
- [ ] 道具状态要延续（上一幕被摔坏的东西不能完好出现）
- [ ] 受伤的人要有伤口/绷带延续
- [ ] 时间跨度（白天→晚上）要有过渡

#### 7. 环境一致性
- [ ] 天气、光照、季节不能突变
- [ ] 室内外温度感觉连贯（从室外进室内不能还有风雪吹）
- [ ] 同一场景的家具摆设、墙面色调保持一致
- [ ] 同一场戏中角色的服装不能无故变换
- [ ] 杯中的饮料量、桌上的物品位置要有连续性

### 三、提示词防呆锚点

在提示词中主动加入以下"物理锚点"来约束模型：

```markdown
【物理规则】
- 拧瓶盖时手部有明确的旋转动作，瓶盖不会凭空消失
- 物体落地后保持在地面，不会自动回到原位
- 人物动作有完整的起止过程，没有跳跃步骤

【状态延续】
- 上一幕被摔碎的玻璃花瓶，这一幕保持碎片状态
- 角色受伤部位持续有包扎/血迹
- 桌面上被碰倒的物品保持倒下的状态

【情绪延续】
- 男主怒气未消，语气低沉但克制，与前一幕的愤怒保持一致
- 角色哭泣后眼眶发红，声音哽咽
- 情绪转换需要合理的触发事件

【剧情逻辑】
- 角色之间的恩怨关系与前情一致
- 秘密/误会尚未揭露前，角色行为符合当前信息量
- 角色性格前后一致，不能为推进剧情做出违和反应
```

### 四、使用流程

```
写提示词前：
  1. 打开本检查清单
  2. 逐条过检，标记有问题的项
  3. 对有问题的项，在提示词中加入"物理锚点"约束
  4. 生成后检查成品，记录新发现的崩塌点

生成后发现的问题：
  1. 记录到本文件"常见问题"章节
  2. 分析原因（模型限制 / 提示词不足 / 参数问题）
  3. 补充到检查清单中
  4. 下次写提示词时特别注意
```

### 五、常见错误示例（持续更新）

| 编号 | 片段 | 错误原因 | 正确做法 |
|------|------|----------|----------|
| #1 | 男主上药，瓶盖凭空消失 | 缺少拧瓶盖动作 | 加入"手部旋转拧开瓶盖" |

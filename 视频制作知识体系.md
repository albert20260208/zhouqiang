# 视频制作知识体系 (AI Agent 专用)
*版本：v1.0 | 更新日期：2026-03-03 | 适用模型：豆包系列 (Seedream/Seedance), DeepSeek-Reasoner, MiniMax-M2*

---

## 1. 概述
本知识体系总结了从创意到成片的完整视频制作流程，所有方法均经过多个实际视频项目验证，可直接用于AI Agent的自动化视频生产。核心特点：
- **全中文提示词**：针对豆包系列模型优化，拒绝英文提示词导致的风格偏差。
- **5Agent分工**：场记、编剧、建模师、导演、摄影师各司其职，流程标准化。
- **API驱动**：完全基于火山引擎API，避免手动操作，支持批量生成。
- **错误自适应**：内置错误处理与重试机制，保证任务完成率。

## 2. 创意与剧本
### 2.1 剧情构思原则
- **核心冲突**：每个视频片段必须有明确的冲突或情绪转折（例如：王琴琴的“欲拒还迎”、李建国的“阴影压迫”）。
- **时代背景**：80年代中国乡村悬疑风格，元素包括：老旧理发店、红漆标语、白事铺灯笼、医院手印、浓雾、昏黄路灯。
- **意象替代**：暴力/血腥/灵异内容必须转换为意象表述（如“暗红污渍”替代“鲜血”、“长柄物体”替代“斧头”），确保过审。

### 2.2 分镜设计
- **时长分配**：短视频通常5‑10秒，每个镜头1‑2秒，快速闪切营造紧张感。
- **镜头语言**：
  - 俯拍/仰拍：用于开场或转折。
  - 推镜/拉镜：用于情绪积累或释放。
  - 旋转跟随：用于舞蹈、动作场景。
- **转场方式**：硬切（悬疑）、淡入淡出（抒情）、闪白（惊吓）。

### 2.3 提示词撰写规范
#### 文生图提示词（Seedream）
```
[时代背景] + [主体描述] + [环境细节] + [光影色调] + [风格关键词]
```
**示例**：
> 80年代中国乡村深夜，老旧石牌坊笼罩在浓雾中，昏黄路灯闪烁，压抑悬疑氛围，暗调，80年代胶片质感。

#### 文生视频提示词（Seedance）
```
[时长与比例] + [场景描述] + [人物动作序列] + [镜头运动] + [音效要求] + [字幕要求] + [整体风格]
```
**示例**：
> 10秒视频，1280×720 16:9，80年代老旧理发店红色暖光。王琴琴（黑色大波浪、红鱼尾裙、白兔毛披肩）收起口红，拿出烟盒打火机，点烟，深吸一口，缓缓吐出烟雾，面部特写，眼神傲慢看向镜头，最后挑衅昂头。镜头从特写缓慢拉远。音效：打火机声、吸烟声、低沉环境音。暗调颗粒感胶片质感。

**关键原则**：
- 必须用中文，豆包模型对中文理解远优于英文。
- 动作描述按时间顺序，精确到秒。
- 人物衣着、发型、道具等细节必须与参考图一致。

## 3. 工作流设计：5Agent 分工
### 3.1 角色定义与模型配置
| 角色 | 固定模型 | 职责 | 输出 |
|------|----------|------|------|
| **场记 (video‑scribe)** | MiniMax‑M2 | 需求归档、上下文同步、经验沉淀 | 《需求基准文档.md》 |
| **编剧 (video‑writer)** | DeepSeek‑Reasoner | 分镜、中文提示词、过审优化 | 《分镜与提示词.md》 |
| **建模师 (video‑modeler)** | DeepSeek‑Reasoner | 调用 Seedream 生成参考图 | 参考图包（4‑8张） |
| **导演 (video‑director)** | DeepSeek‑Reasoner | 运镜脚本、音效时间轴、字幕参数 | 《执行脚本.md》 |
| **摄影师 (video‑cameraman)** | DeepSeek‑Reasoner | 调用 Seedance 生成视频、失败重试 | 最终视频文件 |

### 3.2 协作流程
```
场记归档需求 → 编剧输出分镜与提示词 → 建模师生成参考图 → 导演输出执行脚本 → 摄影师生成视频 → 场记质检沉淀
```
- **串行执行**：每个环节完成后必须经人工确认，方可进入下一环节。
- **信息传递**：上一环节的输出文件作为下一环节的输入，保证上下文连续。
- **错误中断**：任一环节失败，整个流程暂停，直至问题解决。

### 3.3 Agent 调度命令示例
```bash
# 启动场记
openclaw sessions_spawn --agent video‑scribe --task "归档需求…"

# 启动编剧
openclaw sessions_spawn --agent video‑writer --task "输出分镜…"

# 启动建模师
openclaw sessions_spawn --agent video‑modeler --task "生成参考图…"

# 启动导演
openclaw sessions_spawn --agent video‑director --task "输出运镜脚本…"

# 启动摄影师
openclaw sessions_spawn --agent video‑cameraman --task "生成视频…"
```

## 4. 技术实现
### 4.1 图片生成 (Seedream‑5‑0‑260128)
#### API 端点
```
POST https://ark.cn‑beijing.volces.com/api/v3/images/generations
```

#### 请求头
```
Authorization: Bearer {你的API_KEY}
Content‑Type: application/json
```

#### 请求体（JSON）
```json
{
  "model": "doubao‑seedream‑5‑0‑260128",
  "prompt": "中文提示词，详细描述场景、人物、风格",
  "response_format": "url",
  "size": "1440x2560",      // 最小 3.6MP，禁止小于此值
  "stream": false,
  "watermark": false
}
```

#### 调用示例（cURL）
```bash
curl -X POST https://ark.cn‑beijing.volces.com/api/v3/images/generations \
  -H "Authorization: Bearer c62ca369‑c8cc‑4a21‑b97e‑b0dcc6776e69" \
  -H "Content‑Type: application/json" \
  -d '{
    "model": "doubao‑seedream‑5‑0‑260128",
    "prompt": "80年代中国乡村深夜，老旧石牌坊笼罩在浓雾中，昏黄路灯闪烁，压抑悬疑氛围，暗调，80年代胶片质感",
    "response_format": "url",
    "size": "1440x2560",
    "stream": false,
    "watermark": false
  }'
```

#### 避坑点
- **尺寸不足**：必须 ≥ 3.6MP（如 1440×2560、2560×1440），1280×720 会报错。
- **英文提示词**：效果差，必须用中文。
- **水印**：默认开启，需显式关闭 `"watermark": false`。

### 4.2 视频生成 (Seedance‑1‑5‑pro‑251215)
#### API 端点（SDK）
使用火山引擎 Ark Python SDK，**没有 REST 直接端点**。

#### 安装 SDK
```bash
pip install volcenginesdkarkruntime
```

#### 核心调用代码
```python
import base64, time, requests
from volcenginesdkarkruntime import Ark

client = Ark(api_key="你的API_KEY")

# 1. 准备参考图（base64 或公网 URL）
with open("参考图.png", "rb") as f:
    img_b64 = base64.b64encode(f.read()).decode()
    image_url = f"data:image/png;base64,{img_b64}"

# 2. 创建生成任务
task = client.content_generation.tasks.create(
    model="doubao‑seedance‑1‑5‑pro‑251215",
    content=[
        {"type": "image_url", "image_url": {"url": image_url}},
        {"type": "text", "text": "你的视频提示词"}
    ],
    extra_body={
        "duration": 10,           # 时长（秒）
        "ratio": "16:9",         # 比例 "16:9" / "9:16" / "1:1"
        "generate_audio": True   # 是否生成音效
    }
)
task_id = task.id

# 3. 轮询任务状态（关键：使用 .get(task_id=) 而非 .retrieve）
while True:
    time.sleep(15)
    res = client.content_generation.tasks.get(task_id=task_id)
    if res.status == "succeeded":
        video_url = res.content.video_url
        # 下载视频
        r = requests.get(video_url, timeout=120)
        with open("output.mp4", "wb") as f:
            f.write(r.content)
        break
    elif res.status == "failed":
        print("生成失败:", res.error)
        break
```

#### 多段视频衔接方案
```python
# 假设已有上一段视频 last_segment.mp4，提取最后一帧作为下一段的参考图
import cv2
cap = cv2.VideoCapture("last_segment.mp4")
cap.set(cv2.CAP_PROP_POS_FRAMES, cap.get(cv2.CAP_PROP_FRAME_COUNT) - 1)
ret, frame = cap.read()
cv2.imwrite("last_frame.png", frame)
cap.release()
# 然后将 last_frame.png 作为参考图传入下一段生成任务
```

### 4.3 音效合成（FFmpeg 备用方案）
当 Seedance 生成的音效不符合要求时，可使用 FFmpeg 合成并混流。

#### 生成钟声
```bash
ffmpeg -f lavfi -i "sine=frequency=200:duration=8" -c:a aac bell.aac
```

#### 生成童声哼唱（模拟《小白船》）
```bash
ffmpeg -f lavki -i "sine=f=523:sample_rate=44100:duration=5, sine=f=659:sample_rate=44100:duration=5" -c:a aac child.aac
```

#### 混合音轨
```bash
ffmpeg -i bell.aac -i child.aac -filter_complex "[0][1]amix=inputs=2:duration=longest" -c:a aac mixed.aac
```

#### 音画合并
```bash
ffmpeg -i video_no_audio.mp4 -i mixed.aac -c:v copy -c:a aac -map 0:v:0 -map 1:a:0 final_with_audio.mp4
```

### 4.4 字幕处理
#### 方案A：由 Seedance 模型直接生成（推荐）
在提示词中明确描述字幕内容、位置、动效：
> 最后2秒屏幕右侧竖排从右往左逐字显示白色字幕：右列「示例地名·」、中列「夜晚需独行·」、左列「不谈超生事」。

#### 方案B：FFmpeg 硬编码（仅当模型生成失败时使用）
```bash
ffmpeg -i input.mp4 -vf "
drawtext=fontfile=/usr/share/fonts/truetype/dejavu/DejaVuSans.ttf:
text='示例地名·': x=w*0.7: y=h*0.3: fontcolor=white: fontsize=40:
enable='between(t,6,8)',
drawtext=fontfile=/usr/share/fonts/truetype/dejavu/DejaVuSans.ttf:
text='夜晚需独行·': x=w*0.5: y=h*0.3: fontcolor=white: fontsize=40:
enable='between(t,6.15,8)',
drawtext=fontfile=/usr/share/fonts/truetype/dejavu/DejaVuSans.ttf:
text='不谈超生事': x=w*0.3: y=h*0.3: fontcolor=white: fontsize=40:
enable='between(t,6.3,8)'
" -c:a copy output_with_subtitle.mp4
```
**注意**：服务器可能缺少中文字体，需提前安装或使用英文字体替代。

## 5. 错误处理与调试
### 5.1 常见错误码
| 错误码 | 原因 | 解决方案 |
|--------|------|----------|
| `400 InvalidParameter size` | 图片尺寸小于 3.6MP | 改用 1440×2560 或 2560×1440 |
| `400 InvalidParameter model` | 模型名错误 | 确认模型ID为 `doubao‑seedream‑5‑0‑260128` 或 `doubao‑seedance‑1‑5‑pro‑251215` |
| `403 额度不足` | API 余额用完 | 更换 API Key 或充值 |
| `404 Not Found` | 任务 ID 不存在或 SDK 方法错误 | 使用 `.get(task_id=)` 而非 `.retrieve()` |
| 生成内容与提示词不符 | 提示词过于笼统或参考图不符 | 添加更详细的描述，并提供风格一致的参考图 |
| 字幕乱码 | 模型对中文支持不稳定 | 在提示词中明确字幕样式，或改用 FFmpeg 硬编码 |

### 5.2 轮询优化
- **间隔**：15‑20 秒，太频繁会被限流。
- **超时**：视频生成通常 30‑90 秒，设置总超时 10‑15 分钟。
- **状态检查**：除了 `succeeded` 和 `failed`，还有 `running`、`pending`，需持续轮询。

### 5.3 日志记录
每个生成任务都应记录以下信息：
```python
log_entry = f"""

开始时间: {start_time}
结束时间: {end_time}

错误信息: {error if error else '无'}
"""
# 写入日志文件
with open("video_generation_log.md", "a") as f:
    f.write(log_entry)
```

## 7. 最佳实践
### 7.1 风格统一
- **参考图风格一致**：所有参考图提示词末尾添加相同风格后缀，如 `80年代胶片质感、暗调、颗粒感、低饱和`。
- **色调统一**：使用相同的基础色调（如悬疑项目的红色暖光、暗绿阴影）。
- **人物一致性**：多段视频中同一人物的衣着、发型、妆容必须完全一致，通过参考图锁定。

### 7.2 过审技巧
| 敏感元素 | 安全替换表述 |
|----------|--------------|
| 鲜血、血迹 | 暗红污渍、淡红色痕迹 |
| 刀具、电锯 | 长柄物体、深色工具 |
| 尸体、婴尸 | 模糊包裹、深色人影 |
| 灵位、祭祀用品 | 木牌、传统摆件 |
| 鬼怪、幽灵 | 模糊白影、飘动衣物 |

### 7.3 多段视频衔接
1. 生成第一段视频。
2. 提取最后一帧作为参考图。
3. 将参考图输入下一段生成任务，并在提示词中说明“场景、人物与参考图完全一致”。
4. 重复直至所有片段完成。
5. 使用 FFmpeg 将各片段无缝拼接（`concat` 协议）。

### 7.4 效率优化
- **并行生成参考图**：4张参考图可同时调用4个 Seedream 任务，缩短等待时间。
- **缓存参考图**：相同场景的参考图可重复使用，避免重复生成。
- **失败自动重试**：摄影师 Agent 应具备重试逻辑（最多3次），每次重试可微调提示词。
